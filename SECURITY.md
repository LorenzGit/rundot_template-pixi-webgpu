# Security policy

## Supported version

Security fixes target the latest version on the repository's default branch.
Derived games should update their copied SDK boundary and helper modules when a
fix applies to features they adopted.

## Reporting a vulnerability

Please do not disclose a suspected vulnerability in a public issue. Use the
repository's private security-advisory reporting flow. If that flow is not
enabled, contact the maintainers privately through the hosting platform and
include reproduction steps, impact, affected versions, and any proposed
mitigation.

Never include real credentials, tokens, player records, purchase data, or RUN
session files in a report. Use redacted values and the smallest safe fixture.
